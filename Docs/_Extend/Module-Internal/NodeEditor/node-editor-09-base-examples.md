# Examples

---

## Airtable to Layout Flow

---

This example demonstrates a typical use case:

1. AirtableTable node loads schema
2. AirtableEntry node provides content
3. LayoutGenerator node creates page layout
4. UISection nodes render components
5. AIFlowGenerator enhances structure

No hardcoded UI or logic is required.

---

## Example: Menu Structure Flow

A menu can be generated by:

- AirtableTable (navigation data)
- Condition nodes (visibility rules)
- MenuGenerator node (hierarchy creation)
- Output binding to UI

Menus become data-driven and AI-augmentable.
