# UI Prefix System

The filename prefix convention and the actual prefixes discovered in this codebase.

---
## Prefix list

- `D_` — not observed as a `.razor` filename prefix in this scan
- `F_` — discovered (example: `Shared/_Core/02_Fields/F_Text_Url.razor`)
- `P_` — discovered (example: `Shared/_Editor/03_Pointers/P_LogCatcher.razor`)
- `A_` — discovered (example: `Shared/_Editor/04_Actions/A_SplitViewCenter.razor`)
- `N_` — discovered (example: `Shared/_Sites/Nav/Section/N_Section.razor`)
- `C_` — discovered (example: `Shared/_Editor/06_Components/C_DividerSticky.razor`)
- `W_` — discovered (example: `Shared/_Editor/07_Widgets/W_Console.razor`)
- `V_` — discovered (example: `Shared/_Editor/08_View/V_Console.razor`)
- `PNL_` — discovered (example: `Shared/_Editor/09_Panels/PNL_Website.razor`)


## Roles

- Prefixes communicate intent quickly and allow folder-agnostic searches.
- Prefixes should align with the numbered layer folder role.
